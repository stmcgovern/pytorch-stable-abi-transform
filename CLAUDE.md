# stable-abi-transform

Clang LibTooling source-to-source rewriter that transforms PyTorch C++ extension code from the unstable internal ABI to the stable ABI. Generic ecosystem tool — works on any PyTorch C++ consumer.

## Build

```bash
dnf install -y clang-devel llvm-devel ninja-build  # Fedora 41, LLVM 19
cmake -GNinja -B build -S . && ninja -C build -j30
```

Requires: LLVM 19, clang-cpp, g++-13, C++20.

## Usage

```bash
TOOL=build/stable-abi-transform
PYTORCH=/path/to/pytorch  # source tree or install root

# Audit (report what needs to change)
$TOOL --mode=audit file.cu -- -std=c++20 -resource-dir /usr/lib/clang/19 \
  -I$PYTORCH/torch/csrc/api/include -I$PYTORCH -I$PYTORCH/torch/include \
  -I/usr/local/cuda/include

# Rewrite (transform in-place, then auto-verify)
$TOOL --mode=rewrite --pytorch-root=$PYTORCH file.cu -- [same flags]

# Verify: compile-based (tries to compile with only stable headers)
$TOOL --mode=verify --pytorch-root=$PYTORCH \
  --extra-includes=/path/to/project/csrc file.cu -- -std=c++20

# Verify: regex-based (legacy fallback, no PyTorch tree needed)
$TOOL --mode=verify --verify-method=regex file.cu -- -std=c++20
```

### Key flags

| Flag | Description |
|------|-------------|
| `--pytorch-root` | Path to PyTorch source/install root (required for compile-verify) |
| `--extra-includes` | Project-specific -I paths (e.g., the project's own csrc/) |
| `--cuda-include` | CUDA include path (auto-detected from /usr/local/cuda/include) |
| `--verify-method` | "compile" (default) or "regex" |
| `--format` | "text" (default) or "json" |
| `--config` | Path to YAML config file (default: `.stable-abi.yaml` if present) |
| `--init-config` | Generate a starter `.stable-abi.yaml` and exit |
| `--project-root` | Process all C++/CUDA files under this directory |

### YAML config

Instead of passing flags on every invocation, create `.stable-abi.yaml`:

```bash
$TOOL --init-config   # generates starter config
$TOOL                 # auto-loads .stable-abi.yaml
```

Config fields mirror CLI flags: `mode`, `format`, `pytorch_root`, `project_root`, `verify_method`, `cuda_include`, `extra_includes`, `sources`.

## Architecture

- `src/main.cpp` — CLI, mode dispatch, ClangTool configuration
- `src/StableAbiAction.{h,cpp}` — FrontendAction + ASTConsumer wiring
- `src/AstCallbacks.{h,cpp}` — AST matchers for types, data_ptr, methods, free funcs
- `src/PreprocessorCallbacks.{h,cpp}` — include and macro rewrites
- `src/Reporter.{h,cpp}` — finding accumulation, text/JSON output
- `src/TransformerRules.{h,cpp}` — AST-based rewrite rules (Clang Transformer)
- `src/Verifier.{h,cpp}` — compile-based and regex-based post-rewrite verification
- `src/Config.{h,cpp}` — YAML config file support
- `src/Rules.h` — Transformation rules. Type/macro/shorthand rules are auto-generated by gen_rules.py. Include rules are hand-maintained (stable headers don't encode which unstable headers map to them).
- `scripts/gen_rules.py` — derives Rules.h from PyTorch stable ABI headers

### Compile-based verification (shadow include tree)

The `--mode=verify` with `--verify-method=compile` creates a temporary shadow include tree containing symlinks to only the stable PyTorch headers:

```
$TMP/torch/csrc/stable/               → $PYTORCH/torch/csrc/stable/
$TMP/torch/csrc/inductor/aoti_torch/  → $PYTORCH/torch/csrc/inductor/aoti_torch/
$TMP/torch/headeronly/                → $PYTORCH/torch/include/torch/headeronly/
```

Then compiles the file with `-I $TMP` only. If it compiles, the file uses only stable ABI. If not, the compiler errors ARE the gap analysis — precise, zero false positives, zero false negatives.

This is sound because stable headers form a closed dependency set: they only include paths under `torch/csrc/stable/`, `torch/csrc/inductor/aoti_torch/`, and `torch/headeronly/`. Verified by grep — zero references to `c10/`, `ATen/`, or `torch/csrc/api/`.

## Regenerating rules

```bash
python3 scripts/gen_rules.py /path/to/pytorch src/Rules.h
```

Parses `torch/csrc/stable/` and `torch/headeronly/` to extract the complete stable API surface.

## Adding rules

### Which rule category?

- **Type/namespace rename** (e.g. `at::Tensor` -> `torch::stable::Tensor`): add to `type_rules` list in `gen_rules.py`
- **Macro rename** (e.g. `TORCH_CHECK` -> `STD_TORCH_CHECK`): add to `macro_rules` list in `gen_rules.py`
- **Comparison macro** (`TORCH_CHECK_EQ`, etc.): add to `comparison_rules` in `gen_rules.py`
- **Dispatch macro** (`AT_DISPATCH_*` -> `THO_DISPATCH_V2`): add to `dispatch_rules` in `gen_rules.py`
- **Method -> free function** (e.g. `tensor.clone()` -> `torch::stable::clone(tensor)`): auto-derived from ops.h parsing — add the function to ops.h in PyTorch, then regenerate
- **Method rename** (e.g. `.dtype()` -> `.scalar_type()`): add to `kMethodRenameRules` in `gen_rules.py`
- **Include mapping** (e.g. `torch/torch.h` -> `torch/csrc/stable/tensor.h`): add to `include_rules` in `gen_rules.py`
- **Complex AST pattern** (multi-node match, conditional rewrite): add a new `addXxxRule()` function in `TransformerRules.cpp`

### Adding a test

1. Create `test/inputs/yourtest.cpp` with the unstable code
2. Create `test/expected/yourtest.cpp` with the expected rewritten output
3. Run `RESOURCE_DIR=/usr/lib/clang/19 bash test/run_tests.sh`

```bash
# 1. Edit rules

# 2. Regenerate Rules.h

```

## PyTorch Stable ABI Header Layout

### Three-layer architecture

1. **torch/headeronly/** — Pure C++ types, zero libTorch dependency
2. **torch/csrc/stable/** — C++ wrappers around stable C APIs
3. **torch/csrc/inductor/aoti_torch/c/** — C interface to libTorch internals (opaque handles)

Stable headers NEVER include unstable code (at::, c10::, torch/csrc/api/).
They delegate to the C shim via opaque pointers (AtenTensorOpaque*, DeviceGuardHandle, etc.).

### torch/csrc/stable/ (12 headers)

| Header | Provides |
|--------|----------|
| tensor.h | Public entry point (includes tensor_struct.h + tensor_inl.h) |
| tensor_struct.h | stable::Tensor class — dim, numel, size, stride, scalar_type, device, is_contiguous, element_size, mutable_data_ptr, const_data_ptr |
| tensor_inl.h | Tensor method implementations (needs stableivalue_conversions.h) |
| device.h | Public entry point (includes device_struct.h + device_inl.h) |
| device_struct.h | stable::Device class |
| device_inl.h | Device method implementations |
| ops.h | 22 stable ops: empty_like, fill_, narrow, new_empty, new_zeros, pad, amax, transpose, zero_, copy_, clone, flatten, unsqueeze, squeeze, select, matmul, reshape, view, from_blob, to, contiguous, sum, subtract, full, empty, parallel_for, get_num_threads |
| accelerator.h | DeviceGuard, Stream, getCurrentStream, getCurrentDeviceIndex (min PyTorch 2.9) |
| library.h | STABLE_TORCH_LIBRARY, TORCH_BOX, StableLibrary for custom op registration |
| macros.h | STD_CUDA_CHECK, STD_CUDA_KERNEL_LAUNCH_CHECK (min PyTorch 2.10) |
| stableivalue_conversions.h | from<T>() / to<T>() between C++ and StableIValue |
| version.h | TORCH_TARGET_VERSION, TORCH_FEATURE_VERSION |

### torch/headeronly/ (37 headers)

| Directory | Key Headers |
|-----------|-------------|
| core/ | ScalarType.h, DeviceType.h, Layout.h, MemoryFormat.h, Dispatch.h, TensorAccessor.h |
| util/ | Half.h, BFloat16.h, Float8_*.h, Exception.h (STD_TORCH_CHECK), HeaderOnlyArrayRef.h |
| macros/ | Macros.h, Export.h |
| cpu/vec/ | intrinsics.h, vec_half.h |

### Note on include path isolation

`-I $PYTORCH` exposes both stable AND unstable headers (c10/, ATen/, torch/csrc/api/ are all under the same root). The compile-based verifier solves this with the shadow include tree — symlinks to only the stable subdirectories. This is why `--pytorch-root` is required for compile-verify.

### CUDA stream handling

The rewriter uses `aoti_torch_get_current_cuda_stream()` (the C shim) rather than `torch::stable::accelerator::getCurrentStream()` because the C++ `Stream` class doesn't expose the raw `cudaStream_t` needed for CUDA kernel launches. The C shim is stable ABI (behind `#ifdef USE_CUDA` in `shim.h`). The compile verifier passes `-DUSE_CUDA` when CUDA is available. CUDA-related include replacements automatically emit `#include <cuda_runtime.h>` for `cudaStream_t`.
